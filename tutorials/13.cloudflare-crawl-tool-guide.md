## 1. What This Tool Does & Why It Matters

### The Problem It Solves

Your AI CLI agent has a critical blind spot: it can only reason over content that exists in its context window. When a developer asks "how does this library's API work?" or "what changed in the latest release?", the agent either halts or hallucinates — because it cannot fetch live web content.

A production-grade crawl tool closes this gap. It gives your agent the ability to **crawl a starting URL, follow links across the site, and return clean content** (Markdown, HTML, or structured JSON) ready to be injected into the agent's context or stored in a knowledge base.

### Why Cloudflare's API vs. Rolling Your Own

| Concern | DIY Scraper | Cloudflare Browser Rendering |
|---------|:-----------:|:----------------------------:|
| JavaScript-rendered SPAs | ❌ Requires Puppeteer/Playwright setup | ✅ Headless browser built-in |
| Bot detection bypass | ❌ Constant maintenance | ✅ Runs from Cloudflare's network |
| robots.txt compliance | ❌ Manual implementation | ✅ Automatic, audit-ready |
| Caching | ❌ Build your own | ✅ R2-backed, configurable TTL |
| Async job management | ❌ Build your own queue | ✅ 7-day job lifecycle built-in |
| Scale (up to 100k pages) | ❌ Infrastructure overhead | ✅ Managed at Cloudflare scale |

### What It Enables for Your Agent

- **Documentation crawl** — "Fetch the entire Stripe API docs as Markdown and build a knowledge base"
- **Competitive research** — "Crawl competitor.com/pricing and extract all plan details as structured JSON"
- **Live codebase reference** — "Crawl the React 19 migration guide and summarise the breaking changes"
- **Authenticated site crawl** — "Fetch all pages from our internal docs behind OAuth"

## 2. Architecture Overview

### How the Two-Step API Works

The Cloudflare crawl API is **intentionally asynchronous**. It never blocks waiting for a full crawl — instead it issues a job ID immediately and lets you poll for completion. This is the right design for crawling hundreds of pages but requires your tool to manage the job lifecycle correctly.

```
Your Agent Request
        │
        ▼
┌───────────────────────────────────────────┐
│         CRAWL TOOL (your code)            │
│                                           │
│  1. Build config from agent parameters    │
│  2. POST /crawl  → receive job_id         │
│  3. Poll GET /crawl/{job_id} with backoff │
│  4. Paginate results via cursor           │
│  5. Normalise and return to agent         │
└───────────────────────────────────────────┘
        │
        ▼
┌───────────────────────────────────────────┐
│    Cloudflare Browser Rendering API       │
│                                           │
│  • Headless browser farm                  │
│  • URL discovery (sitemap + link follow)  │
│  • robots.txt enforcement                 │
│  • R2 cache layer                         │
│  • Workers AI for JSON extraction         │
└───────────────────────────────────────────┘
```

### Tool Module Structure

```
src/tools/
└── web-crawl/
    ├── index.ts          ← Tool registration (name, schema, execute)
    ├── client.ts         ← Cloudflare API client (POST, GET, DELETE)
    ├── poller.ts         ← Async polling with exponential backoff
    ├── paginator.ts      ← Cursor-based result pagination
    ├── normaliser.ts     ← Convert raw records → agent-ready content
    ├── config.ts         ← Crawl strategy presets and validation
    └── types.ts          ← TypeScript interfaces for all API shapes
```

## 3. Prerequisites & Setup

### Cloudflare Account Requirements

You need the following before writing a single line:

1. **Cloudflare account** with Workers enabled
2. **Account ID** — Found in the Cloudflare dashboard right sidebar under "Account ID"
3. **API Token** — Go to `dash.cloudflare.com → My Profile → API Tokens → Create Token`
   - Select the "Browser Rendering" permission template
   - Scope to your account
   - Set an expiry (90 days recommended; rotate regularly)

## 4. Core API Mechanics — How the Crawl Endpoint Works

### The Endpoint

```
POST  https://api.cloudflare.com/client/v4/accounts/{account_id}/browser-rendering/crawl
GET   https://api.cloudflare.com/client/v4/accounts/{account_id}/browser-rendering/crawl/{job_id}
DELETE https://api.cloudflare.com/client/v4/accounts/{account_id}/browser-rendering/crawl/{job_id}
```

### Step 1 — Initiate the Crawl (POST)

Send a POST with your `url` and options. The response returns immediately with a `job_id`. The crawl runs in the background.

```
Request:  POST /crawl  { "url": "https://docs.example.com", "limit": 100 }
Response: { "success": true, "result": "c7f8s2d9-a8e7-4b6e-8e4d-3d4a1b2c3f4e" }
```

### Step 2 — Poll for Status (GET)

Poll using the job ID. The `status` field tells you the state:

| Status | Meaning | Your Action |
|--------|---------|-------------|
| `running` | Still crawling | Wait and poll again |
| `completed` | All done | Fetch full results |
| `errored` | Crawl failed | Log error, retry or surface to agent |
| `cancelled_due_to_timeout` | Exceeded 7-day max | Crawl was too large — reduce scope |
| `cancelled_due_to_limits` | Hit account limits | Upgrade plan or reduce `limit` |
| `cancelled_by_user` | You cancelled it | Expected — no action needed |

**Critical optimisation:** While polling, always append `?limit=1` to keep the response lightweight. You only need the `status` — not the full set of records.

```
GET /crawl/{job_id}?limit=1
→ { "result": { "status": "running", "finished": 23, "total": 100 } }
```

### Step 3 — Fetch Paginated Results (GET without limit=1)

Once `status === "completed"`, fetch the full results. If the total response would exceed 10 MB, the API includes a `cursor` value. Pass it back as a query parameter to get the next page.

```
GET /crawl/{job_id}                          → page 1 (includes cursor: 50)
GET /crawl/{job_id}?cursor=50                → page 2 (includes cursor: 100)
GET /crawl/{job_id}?cursor=100               → page 3 (no cursor = last page)
```

Each record in `records[]` contains:

```json
{
  "url": "https://docs.example.com/api",
  "status": "completed",
  "markdown": "# API Reference\nAll endpoints...",
  "html": "<h1>API Reference</h1>...",
  "metadata": {
    "status": 200,
    "title": "API Reference",
    "url": "https://docs.example.com/api"
  }
}
```

### URL Discovery Behaviour

When `source: "all"` (default), the crawler discovers URLs in this order:

1. **Starting URL** — The URL you provided
2. **Sitemap links** — Parsed from `/sitemap.xml` and any linked sitemaps
3. **Page links** — `<a href>` tags scraped from crawled pages

Use `source: "sitemaps"` for large documentation sites where the sitemap is reliable. Use `source: "links"` when there is no sitemap and you want to follow page links only.

### Step 2 — Build the API Client

Create `src/tools/web-crawl/client.ts`:

```typescript
import type { WebCrawlInput, CrawlJobResult } from './types.js'

const BASE_URL = 'https://api.cloudflare.com/client/v4/accounts'

function getCredentials() {
  const accountId = process.env.CLOUDFLARE_ACCOUNT_ID
  const apiToken = process.env.CLOUDFLARE_API_TOKEN

  if (!accountId || !apiToken) {
    throw new Error(
      'Missing Cloudflare credentials. Set CLOUDFLARE_ACCOUNT_ID and CLOUDFLARE_API_TOKEN environment variables.'
    )
  }

  return { accountId, apiToken }
}

// ─── Initiate Crawl ───────────────────────────────────────────────────────────

export async function startCrawl(input: WebCrawlInput): Promise<string> {
  const { accountId, apiToken } = getCredentials()

  const body = buildRequestBody(input)

  const response = await fetch(`${BASE_URL}/${accountId}/browser-rendering/crawl`, {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${apiToken}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify(body),
  })

  if (!response.ok) {
    const error = await response.text()
    throw new Error(`Failed to start crawl: HTTP ${response.status} — ${error}`)
  }

  const data = await response.json() as { success: boolean; result: string }

  if (!data.success) {
    throw new Error(`Cloudflare API returned success:false when starting crawl`)
  }

  return data.result // job ID
}

// ─── Poll Job Status ──────────────────────────────────────────────────────────

export async function getCrawlStatus(jobId: string): Promise<Pick<CrawlJobResult, 'status' | 'total' | 'finished'>> {
  const { accountId, apiToken } = getCredentials()

  const url = `${BASE_URL}/${accountId}/browser-rendering/crawl/${jobId}?limit=1`

  const response = await fetch(url, {
    headers: { Authorization: `Bearer ${apiToken}` },
  })

  if (!response.ok) {
    throw new Error(`Failed to get crawl status: HTTP ${response.status}`)
  }

  const data = await response.json() as { result: CrawlJobResult }
  const { status, total, finished } = data.result

  return { status, total, finished }
}

// ─── Fetch One Page of Results ────────────────────────────────────────────────

export async function getCrawlResults(
  jobId: string,
  options: { cursor?: number; limit?: number; statusFilter?: string } = {}
): Promise<{ result: CrawlJobResult }> {
  const { accountId, apiToken } = getCredentials()

  const params = new URLSearchParams()
  if (options.cursor !== undefined) params.set('cursor', String(options.cursor))
  if (options.limit !== undefined) params.set('limit', String(options.limit))
  if (options.statusFilter) params.set('status', options.statusFilter)

  const query = params.toString()
  const url = `${BASE_URL}/${accountId}/browser-rendering/crawl/${jobId}${query ? `?${query}` : ''}`

  const response = await fetch(url, {
    headers: { Authorization: `Bearer ${apiToken}` },
  })

  if (!response.ok) {
    throw new Error(`Failed to get crawl results: HTTP ${response.status}`)
  }

  return response.json()
}

// ─── Cancel a Running Job ─────────────────────────────────────────────────────

export async function cancelCrawl(jobId: string): Promise<void> {
  const { accountId, apiToken } = getCredentials()

  const response = await fetch(
    `${BASE_URL}/${accountId}/browser-rendering/crawl/${jobId}`,
    {
      method: 'DELETE',
      headers: { Authorization: `Bearer ${apiToken}` },
    }
  )

  if (!response.ok) {
    throw new Error(`Failed to cancel crawl ${jobId}: HTTP ${response.status}`)
  }
}

// ─── Request Body Builder ─────────────────────────────────────────────────────

function buildRequestBody(input: WebCrawlInput): Record<string, unknown> {
  const body: Record<string, unknown> = {
    url: input.url,
    limit: input.limit ?? 100,
    depth: input.depth ?? 10,
    formats: input.formats ?? ['markdown'],
    render: input.render ?? true,
    source: input.source ?? 'all',
  }

  if (input.maxAge !== undefined) {
    body.maxAge = input.maxAge
  }

  if (input.authenticate) {
    body.authenticate = input.authenticate
  }

  if (input.extraHeaders && Object.keys(input.extraHeaders).length > 0) {
    body.setExtraHTTPHeaders = input.extraHeaders
  }

  if (input.rejectResourceTypes && input.rejectResourceTypes.length > 0) {
    body.rejectResourceTypes = input.rejectResourceTypes
  }

  if (input.waitForSelector) {
    body.waitForSelector = {
      selector: input.waitForSelector.selector,
      timeout: input.waitForSelector.timeout,
      visible: true,
    }
  }

  if (input.waitUntil) {
    body.gotoOptions = { waitUntil: input.waitUntil, timeout: 60000 }
  }

  if (input.jsonPrompt && input.formats?.includes('json')) {
    body.jsonOptions = { prompt: input.jsonPrompt }
  }

  const options: Record<string, unknown> = {}
  if (input.includeExternalLinks !== undefined) {
    options.includeExternalLinks = input.includeExternalLinks
  }
  if (input.includeSubdomains !== undefined) {
    options.includeSubdomains = input.includeSubdomains
  }
  if (input.includePatterns && input.includePatterns.length > 0) {
    options.includePatterns = input.includePatterns
  }
  if (input.excludePatterns && input.excludePatterns.length > 0) {
    options.excludePatterns = input.excludePatterns
  }

  if (Object.keys(options).length > 0) {
    body.options = options
  }

  return body
}
```

### Step 3 — Build the Poller with Exponential Backoff

Create `src/tools/web-crawl/poller.ts`:

```typescript
import { getCrawlStatus } from './client.js'
import type { CrawlJobResult } from './types.js'

const TERMINAL_STATUSES = new Set([
  'completed',
  'errored',
  'cancelled_due_to_timeout',
  'cancelled_due_to_limits',
  'cancelled_by_user',
])

interface PollOptions {
  maxAttempts?: number
  initialDelayMs?: number
  maxDelayMs?: number
  onProgress?: (finished: number, total: number) => void
}

/**
 * Polls the crawl job status with exponential backoff until it reaches
 * a terminal state. Returns the final status object.
 *
 * Uses a lightweight ?limit=1 request on every poll to avoid
 * downloading full results prematurely.
 */
export async function pollUntilComplete(
  jobId: string,
  options: PollOptions = {}
): Promise<Pick<CrawlJobResult, 'status' | 'total' | 'finished'>> {
  const maxAttempts = options.maxAttempts ?? parseInt(process.env.CRAWL_MAX_POLL_ATTEMPTS ?? '60')
  const initialDelay = options.initialDelayMs ?? parseInt(process.env.CRAWL_POLL_INTERVAL_MS ?? '5000')
  const maxDelay = options.maxDelayMs ?? 30_000

  let currentDelay = initialDelay
  let consecutiveErrors = 0
  const maxConsecutiveErrors = 5

  for (let attempt = 0; attempt < maxAttempts; attempt++) {
    try {
      const result = await getCrawlStatus(jobId)

      if (options.onProgress) {
        options.onProgress(result.finished, result.total)
      }

      if (TERMINAL_STATUSES.has(result.status)) {
        return result
      }

      // Reset error counter on success
      consecutiveErrors = 0

      // Wait before next poll — capped exponential backoff
      await sleep(currentDelay)
      currentDelay = Math.min(currentDelay * 1.5, maxDelay)
    } catch (error) {
      consecutiveErrors++

      if (consecutiveErrors >= maxConsecutiveErrors) {
        throw new Error(
          `Polling failed ${maxConsecutiveErrors} consecutive times for job ${jobId}. ` +
          `Last error: ${error instanceof Error ? error.message : String(error)}`
        )
      }

      // Back off more aggressively on errors
      await sleep(currentDelay * 2)
    }
  }

  throw new Error(
    `Crawl job ${jobId} did not reach a terminal state after ${maxAttempts} poll attempts. ` +
    `Consider increasing CRAWL_MAX_POLL_ATTEMPTS or checking the Cloudflare dashboard.`
  )
}

function sleep(ms: number): Promise<void> {
  return new Promise((resolve) => setTimeout(resolve, ms))
}
```

### Step 4 — Build the Paginator

Create `src/tools/web-crawl/paginator.ts`:

```typescript
import { getCrawlResults } from './client.js'
import type { CrawlRecord } from './types.js'

const PAGE_SIZE = parseInt(process.env.CRAWL_RESULT_PAGE_SIZE ?? '50')

/**
 * Fetches ALL completed records from a finished crawl job,
 * automatically following cursor pagination until no cursor is returned.
 *
 * Only fetches records with status='completed' to skip disallowed/skipped URLs.
 */
export async function fetchAllResults(
  jobId: string,
  onPage?: (records: CrawlRecord[], page: number) => void
): Promise<CrawlRecord[]> {
  const allRecords: CrawlRecord[] = []
  let cursor: number | undefined = undefined
  let page = 0

  while (true) {
    const response = await getCrawlResults(jobId, {
      cursor,
      limit: PAGE_SIZE,
      statusFilter: 'completed',
    })

    const { records, cursor: nextCursor } = response.result

    if (records && records.length > 0) {
      allRecords.push(...records)
      if (onPage) onPage(records, ++page)
    }

    if (nextCursor === undefined || records.length < PAGE_SIZE) {
      break
    }

    cursor = nextCursor
  }

  return allRecords
}

/**
 * Returns a summary of all URL statuses without downloading full content.
 * Useful for diagnostics and reporting to the user.
 */
export async function fetchStatusSummary(jobId: string): Promise<{
  completed: number
  disallowed: number
  skipped: number
  errored: number
  cancelled: number
}> {
  const statusTypes = ['completed', 'disallowed', 'skipped', 'errored', 'cancelled'] as const
  const counts: Record<string, number> = {}

  for (const status of statusTypes) {
    const response = await getCrawlResults(jobId, { statusFilter: status, limit: 1 })
    counts[status] = response.result.total ?? 0
  }

  return {
    completed: counts.completed ?? 0,
    disallowed: counts.disallowed ?? 0,
    skipped: counts.skipped ?? 0,
    errored: counts.errored ?? 0,
    cancelled: counts.cancelled ?? 0,
  }
}
```

### Step 5 — Build the Normaliser

Create `src/tools/web-crawl/normaliser.ts`:

```typescript
import type { CrawlRecord, CrawlResult, CrawlJobResult } from './types.js'

const MAX_CONTENT_CHARS_PER_PAGE = 50_000
const MAX_TOTAL_CHARS = 400_000

/**
 * Converts raw Cloudflare crawl records into a clean, agent-ready format.
 * Applies character limits per page and across the full result set to
 * prevent context window overflow.
 */
export function normaliseResults(
  jobId: string,
  jobResult: Pick<CrawlJobResult, 'status' | 'total' | 'finished' | 'browserSecondsUsed'>,
  records: CrawlRecord[],
  options: { maxTotalChars?: number; maxCharsPerPage?: number } = {}
): CrawlResult {
  const maxTotal = options.maxTotalChars ?? MAX_TOTAL_CHARS
  const maxPerPage = options.maxCharsPerPage ?? MAX_CONTENT_CHARS_PER_PAGE

  const pages: CrawlResult['pages'] = []
  let totalCharsUsed = 0
  let truncatedPages = 0

  for (const record of records) {
    if (totalCharsUsed >= maxTotal) {
      truncatedPages++
      continue
    }

    const raw = record.markdown ?? record.html ?? (record.json ? JSON.stringify(record.json, null, 2) : '')
    const contentType = record.markdown ? 'markdown' : record.html ? 'html' : 'json'

    let content = raw
    if (content.length > maxPerPage) {
      content = content.slice(0, maxPerPage) + `\n\n[...content truncated at ${maxPerPage} characters]`
    }

    if (totalCharsUsed + content.length > maxTotal) {
      const remaining = maxTotal - totalCharsUsed
      content = content.slice(0, remaining) + '\n\n[...truncated due to total output limit]'
      truncatedPages++
    }

    totalCharsUsed += content.length

    pages.push({
      url: record.url,
      title: record.metadata?.title ?? extractTitleFromContent(content) ?? record.url,
      content,
      contentType,
    })
  }

  const summary = buildSummary(jobId, jobResult, records, pages, truncatedPages)

  return {
    jobId,
    status: jobResult.status,
    pagesFound: jobResult.total,
    pagesCompleted: records.length,
    pagesDisallowed: 0,   // populated separately if needed
    pagesSkipped: 0,
    pagesErrored: 0,
    browserSecondsUsed: jobResult.browserSecondsUsed,
    pages,
    summary,
  }
}

function extractTitleFromContent(content: string): string | null {
  const match = content.match(/^#\s+(.+)/m)
  return match ? match[1].trim() : null
}

function buildSummary(
  jobId: string,
  jobResult: Pick<CrawlJobResult, 'status' | 'total' | 'finished' | 'browserSecondsUsed'>,
  records: CrawlRecord[],
  pages: CrawlResult['pages'],
  truncatedCount: number
): string {
  const lines = [
    `Crawl completed — Job ID: ${jobId}`,
    `Status: ${jobResult.status}`,
    `Pages crawled: ${jobResult.finished} of ${jobResult.total} found`,
    `Pages returned: ${pages.length}${truncatedCount > 0 ? ` (${truncatedCount} truncated due to size limit)` : ''}`,
  ]

  if (jobResult.browserSecondsUsed !== undefined) {
    lines.push(`Browser time used: ${jobResult.browserSecondsUsed.toFixed(1)}s`)
  }

  return lines.join('\n')
}
```

```typescript
  /**
   * For sites behind authentication.
   * Requires authenticate or extraHeaders to be provided.
   */
  authenticated: {
    limit: 100,
    depth: 5,
    formats: ['markdown'],
    render: true,
    waitUntil: 'networkidle2',
    maxAge: 0,   // Never cache authenticated content
  },

  /**
   * For e-commerce product catalogs.
   * Extracts structured JSON using Cloudflare's Workers AI.
   */
  'product-catalog': {
    limit: 50,
    depth: 2,
    formats: ['json'],
    render: true,
    rejectResourceTypes: ['font', 'stylesheet'],
  },
}

/**
 * Apply a named strategy preset to a user input object.
 * User-provided values always take precedence over preset values.
 */
export function applyStrategy(input: WebCrawlInput): WebCrawlInput {
  if (!input.strategy) return input

  const preset = CRAWL_STRATEGIES[input.strategy]
  if (!preset) {
    throw new Error(
      `Unknown strategy "${input.strategy}". Available strategies: ${Object.keys(CRAWL_STRATEGIES).join(', ')}`
    )
  }

  // User-provided values win over preset defaults
  return { ...preset, ...input }
}
```

```typescript
          `Partial results may still be available — fetch them at:\n` +

      process.stdout.write(`[web_crawl] Fetching results...\n`)

        }

      return `Crawl failed: ${message}`

// ─── Output Formatters ────────────────────────────────────────────────────────

function formatCrawlResult(result: CrawlResult): string {
  const lines: string[] = [
    '---',
    result.summary,
    '---',
    '',
  ]

  for (const page of result.pages) {
    lines.push(`## ${page.title}`)
    lines.push(`URL: ${page.url}`)
    lines.push(`Format: ${page.contentType}`)
    lines.push('')
    lines.push(page.content)
    lines.push('')
    lines.push('---')
    lines.push('')
  }

  if (result.pages.length === 0) {
    lines.push('No pages were successfully crawled. Possible reasons:')
    lines.push('• robots.txt is blocking the crawler')
    lines.push('• includePatterns are too restrictive')
    lines.push('• The starting URL has no discoverable links')
  }

  return lines.join('\n')
}

  status: string,
  summary: { completed: number; disallowed: number; skipped: number }

    `• Reduce the "limit" parameter`,
    `• Set render: false for static content (uses Workers, not browser time)`,
    `• Increase maxAge to use cached results`,
    `• Upgrade to Workers Paid plan for higher limits`,
    `See: https://developers.cloudflare.com/browser-rendering/limits/`,
  ].join('\n')
}
```

## 7. Configuration Reference

### All Input Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `url` | string | **required** | Starting URL for the crawl |
| `strategy` | string | — | Preset: `docs`, `spa`, `static`, `authenticated`, `product-catalog` |
| `limit` | number | 100 | Maximum pages to crawl (max: 100,000) |
| `depth` | number | 10 | Maximum link depth from starting URL |
| `formats` | string[] | `['markdown']` | Output formats: `html`, `markdown`, `json` |
| `render` | boolean | `true` | Execute JavaScript (headless browser); `false` = fast HTML fetch |
| `source` | string | `'all'` | URL discovery: `all`, `sitemaps`, `links` |
| `maxAge` | number | 86400 | Cache TTL in seconds; `0` = always fresh |
| `includePatterns` | string[] | — | Wildcard patterns; only matching URLs are crawled |
| `excludePatterns` | string[] | — | Wildcard patterns; matching URLs are skipped |
| `includeSubdomains` | boolean | `false` | Follow links to subdomains |
| `includeExternalLinks` | boolean | `false` | Follow links to other domains |
| `authenticate` | object | — | `{ username, password }` for HTTP basic auth |
| `extraHeaders` | object | — | Custom headers (e.g., API keys, session tokens) |
| `rejectResourceTypes` | string[] | — | Block: `image`, `media`, `font`, `stylesheet`, `script` |
| `waitUntil` | string | — | `networkidle2` for heavy SPAs; `domcontentloaded` for faster loads |
| `waitForSelector` | object | — | `{ selector, timeout }` — wait for a DOM element before scraping |
| `jsonPrompt` | string | — | AI extraction prompt when `formats` includes `json` |

### Pattern Syntax

Cloudflare supports two wildcard types:

| Syntax | Matches | Example |
|--------|---------|---------|
| `*` | Any characters **except** `/` | `**/api/*` matches `/api/users` but not `/api/v1/users` |
| `**` | Any characters **including** `/` | `**/api/**` matches `/api/v1/users` and all sub-paths |

**Pattern priority rule:** `excludePatterns` always wins. If a URL matches both an include and exclude pattern, it is skipped.

## 9. Error Handling & Resilience

### Error Taxonomy

| Error Type | Cause | Recovery |
|-----------|-------|----------|
| `cancelled_due_to_limits` | Account browser time exhausted | Reduce `limit`, use `render: false`, upgrade plan |
| `cancelled_due_to_timeout` | Crawl exceeded 7-day maximum | Smaller scope, add `includePatterns` |
| HTTP 401 | Bad or expired API token | Rotate token in Cloudflare dashboard |
| HTTP 429 | API rate limit | Back off and retry; add exponential delay |
| HTTP 400 | Invalid request body | Check parameter names and types against the API reference |
| Polling timeout | Job running too long | Check `browserSecondsUsed` — site may be very heavy; try `render: false` |
| All URLs disallowed | `robots.txt` blocking | Check target site's robots.txt; crawl is respecting it correctly |
| No records returned | Patterns too restrictive | Try without `includePatterns` first to confirm URLs are discoverable |

### Retry Logic for Transient Failures

The poller already handles transient HTTP errors with backoff. For the initial `POST`, wrap in a retry:

```typescript
async function startCrawlWithRetry(input: WebCrawlInput, maxRetries = 3): Promise<string> {
  let lastError: Error | undefined

  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      return await startCrawl(input)
    } catch (error) {
      lastError = error instanceof Error ? error : new Error(String(error))

      // Don't retry on auth errors or bad request errors
      if (lastError.message.includes('HTTP 401') || lastError.message.includes('HTTP 400')) {
        throw lastError
      }

      if (attempt < maxRetries) {
        const delay = 1000 * Math.pow(2, attempt)
        await new Promise((resolve) => setTimeout(resolve, delay))
      }
    }
  }

  throw lastError
}
```

### Graceful Cancellation on Agent Shutdown

If your REPL process receives `SIGINT` (Ctrl+C) during a crawl, cancel the active job to avoid wasting browser time:

```typescript
let activeCrawlJobId: string | undefined

process.on('SIGINT', async () => {

    } catch {
      // Non-fatal
    }
  }

})

// In your execute() function, set this before polling:

// Clear it when done:

```

## 11. Rate Limits, Costs & Performance

### Cloudflare Limits (April 2026)

| Plan | Browser Time | Notes |
|------|-------------|-------|
| Workers Free | 10 min/day | Very limited — development only |
| Workers Paid | Higher limits | See [limits page](https://developers.cloudflare.com/browser-rendering/limits/) |

### Cost Awareness

- **`render: true`** crawls use a headless browser — billed under Browser Rendering pricing
- **`render: false`** crawls run on Workers — currently **free during beta**; will move to Workers pricing post-beta
- **JSON extraction** uses Workers AI for each page — incurs Workers AI usage on top of crawl costs

**Practical guidance:**

| Site Type | Recommended Setting | Cost Impact |
|-----------|-------------------|-------------|
| Static HTML docs | `render: false` | Free (beta) / Workers pricing |
| SPAs / JS-heavy | `render: true` | Browser Rendering pricing |
| Large static sites | `render: false` + `maxAge: 86400` | Minimised; cache reduces repeat crawls |
| Product catalog with AI | `render: true` + `formats: ["json"]` | Browser + Workers AI |

### Performance Optimisations

1. **Block unnecessary resources** — Always include `rejectResourceTypes: ["image", "media", "font"]` unless you specifically need those resources. This meaningfully speeds up `render: true` crawls.

2. **Use the sitemap source** — For large documentation sites with reliable sitemaps, `source: "sitemaps"` is faster than `source: "links"` because it does not need to scrape each page for further links.

3. **Set a reasonable `maxAge`** — For content that changes weekly, `maxAge: 604800` (7 days) eliminates repeat crawls of the same pages. Use `maxAge: 0` only for real-time monitoring use cases.

4. **Use `depth` before `limit`** — Lowering `depth` is often more precise than lowering `limit`. A `depth: 2` crawl from a docs homepage will stay within the top-level sections; `limit: 50` with high depth may wander deep into irrelevant pages.

## 12. Troubleshooting Reference

### Problem: All URLs Return `"status": "disallowed"`

**Cause:** The target site's `robots.txt` is blocking the Cloudflare Browser Rendering crawler.

**How to investigate:**

```bash
curl https://example.com/robots.txt
```

If the crawl is for a site you own: Add a `robots.txt` exception for the Cloudflare Browser Rendering user agent, or add a WAF skip rule. See [Cloudflare's guide](https://developers.cloudflare.com/browser-rendering/faq/#will-browser-rendering-bypass-cloudflares-bot-protection).

If the site belongs to a third party: The crawl is correctly respecting the site's `robots.txt`. You cannot override this.

---

### Problem: Empty Results Despite `status: completed`

**Diagnosis steps:**

1. First, crawl without any `includePatterns`:
   ```typescript
   { url: "https://example.com", limit: 5, formats: ["markdown"] }
   ```
   If this returns results, your patterns are too restrictive.

2. Check skipped URLs:
   ```typescript
   // Add statusFilter: "skipped" to your results fetch
   getCrawlResults(jobId, { statusFilter: 'skipped' })
   ```

3. The starting URL may not link to other pages. Try `source: "sitemaps"` to discover URLs from the sitemap directly.

---

### Problem: Crawl is Very Slow

Most likely causes and fixes:

| Cause | Fix |
|-------|-----|
| Site has heavy JS bundles | Add `rejectResourceTypes: ["script", "stylesheet"]` |
| Pages include large images/videos | Add `rejectResourceTypes: ["image", "media"]` |
| Site has a `crawl-delay` in `robots.txt` | Crawler respects this — you cannot bypass it |
| `waitUntil: "networkidle2"` is waiting too long | Use `"domcontentloaded"` if you don't need full JS execution |

---

### Problem: Crawl Job Hits Account Limits

```
status: "cancelled_due_to_limits"
```

**Immediate fixes:**

1. Set `render: false` — Workers-based fetches don't consume browser time
2. Set `maxAge: 86400` — Use caching to avoid re-fetching pages
3. Reduce `limit` — Crawl in multiple smaller batches
4. Check if you're on Workers Free plan — upgrade to Paid for production use

---

### Problem: JSON Extraction Returns Nulls

The AI extraction is not finding the fields you specified.

**Fixes:**

1. Make the prompt more specific about where the data appears:
   ```
   "Extract product name from the <h1> tag and price from the element 
   with class 'product-price'. Description is in the first paragraph 
   of the .product-description section."
   ```

2. Use a `response_format` schema to enforce structure:
   ```typescript
   jsonOptions: {
     prompt: "...",
     response_format: {
       type: "json_schema",
       json_schema: {
         name: "product",
         properties: {
           name: "string",
           price: "number"
         }
       }
     }
   }
   ```

3. Ensure `render: true` is set — the content may not be in the initial HTML.
