## Part 1 — Individual Profiles

---

### Vercel AI SDK (`ai`)

**Status:** Production-stable (v6.x) | **License:** Apache 2.0 | **Maintained by:** Vercel  
**Repo:** [github.com/vercel/ai](https://github.com/vercel/ai) | **Weekly Downloads:** ~20M+

#### What It Is

The Vercel AI SDK is the most-used TypeScript AI toolkit in the JavaScript ecosystem — shipped by the team behind Next.js and currently at v6 (released early 2026). It started as a UI-focused streaming library but has steadily grown into a comprehensive agent toolkit covering text generation, structured output, tool calling, multi-step agent loops, MCP integration, speech/audio, embeddings, and image generation.

The core philosophy is "one unified API for every provider." You write `generateText({ model: 'anthropic/claude-opus-4.6', ... })` and it works identically whether you're targeting Anthropic, OpenAI, Google, Mistral, Groq, or 25+ others. Provider switching is a one-liner.

#### Architecture

The SDK is structured in three layers:

**AI SDK Core** (`ai` package) — Framework-agnostic functions: `generateText`, `streamText`, `generateObject`, `streamObject`, `embed`, and `ToolLoopAgent`. These work in any Node.js, Bun, Deno, or edge runtime without React. For a terminal agent this is the only layer you touch.

**AI SDK Providers** (`@ai-sdk/openai`, `@ai-sdk/anthropic`, `@ai-sdk/google`, etc.) — Thin, typed adapter packages for each model provider. Each package exposes a factory function (e.g., `openai('gpt-5.4')`) that returns a `LanguageModel` object consumed by Core functions. You install only the providers you need.

**AI SDK UI** (`@ai-sdk/react`, `@ai-sdk/svelte`, `@ai-sdk/vue`) — React/framework hooks (`useChat`, `useObject`, `useCompletion`) for building streaming chat UIs. Not relevant for terminal agents — you don't need these.

#### Key Features

**`streamText` and `generateText`** are the workhorses. Both accept a `model`, `messages`, `tools`, and optionally a `system` prompt. `streamText` returns an async iterable of text deltas you can pipe to stdout directly — exactly what a terminal agent needs. For a coding agent:

```typescript
import { streamText } from 'ai'
import { anthropic } from '@ai-sdk/anthropic'

const result = streamText({
  model: anthropic('claude-opus-4-6'),
  system: 'You are an expert coding assistant.',
  messages,
  tools: { readFile, writeFile, runShell },
  stopWhen: stepCountIs(10),
})

for await (const chunk of result.textStream) {
  process.stdout.write(chunk)
}
```

**`ToolLoopAgent`** (v6+) is an OOP wrapper around the tool execution loop. It calls the model, executes any tool calls, adds results back to the conversation, and repeats until `stopWhen` is satisfied. Introduced in v6 to replace manual agentic loop management. It also supports `human-in-the-loop` approval — pausing execution before tool calls that have side effects.

**`generateObject` / `streamObject`** — Structured output with Zod schema validation. The model returns a typed object matching your schema, not raw JSON. Useful for parsing LLM output into typed data structures (e.g., file diffs, lint results, dependency lists).

**Language Model Middleware** — Intercept and modify any LLM call: add caching, guardrails, logging, or retry logic without touching your core agent code. Three built-in middleware: `extractReasoningMiddleware` (pulls `<think>` blocks), `simulateStreamingMiddleware` (makes non-streaming providers appear to stream), and `rateLimitMiddleware`.

**MCP Support (full)** — Connect to any MCP server via stdio or SSE, discover its tools, and use them natively. This is critical for a coding agent — you can connect to git, browser, file system, or database MCP servers with no custom wiring.

**Global Provider** — The default provider routes through Vercel's AI Gateway, meaning `generateText({ model: 'openai/gpt-5.4' })` works without installing `@ai-sdk/openai` — the gateway handles authentication. This is optional; you can always import providers directly.

#### Strengths

The ecosystem breadth is unmatched. With over 20 million monthly downloads and adoption by teams ranging from startups to Fortune 500 companies, it's the leading TypeScript toolkit for building AI applications. That means production-tested code, fast bug fixes, extensive examples (100+ templates), and a large community to learn from.

The Node.js terminal quickstart guide is first-class — the official docs have a dedicated Node.js getting-started that builds a streaming CLI agent with tools using nothing but `streamText` and `readline`. No React, no Next.js, no framework required.

AI SDK 6 unifies `generateObject` and `generateText` to enable multi-step tool-calling loops with structured output generation at the end. The `ToolLoopAgent` class is production-hardened for complex autonomous agents.

#### Weaknesses

In the Vercel AI SDK, tools are split across environments — you define a tool on the server where the LLM executes it, and then reimplement the same tool on the client so the UI can interpret and display its behavior. This duplication introduces extra surface area for drift between server logic and client state. (This mostly matters for web apps, not terminal agents — but it's a design smell.)

One weakness of the AI SDK is that the options you pass to the provider are not strongly typed. You can attempt to use structured outputs on a model that doesn't support them, and it won't warn you at compile time.

The Vercel AI Gateway, while optional, is increasingly the recommended default, creating a soft gravitational pull toward Vercel's paid infrastructure. Teams that want zero-dependency on Vercel's servers must explicitly opt out.

#### Terminal Agent Fit: ★★★★★

`streamText` works natively in Node.js terminal contexts. OpenCode (122k stars), which is the most widely deployed JS/TS terminal agent, uses the Vercel AI SDK exclusively. Real-world validation at massive scale.

### TanStack AI (`@tanstack/ai`)

**Status:** Alpha (v0.x) | **License:** MIT | **Maintained by:** TanStack community (Netlify-sponsored)  
**Repo:** [github.com/TanStack/ai](https://github.com/TanStack/ai) | **Weekly Downloads:** Early stage  
**Launched:** December 3, 2025 | **GitHub Stars:** ~2.2k (April 2026)

#### What It Is

TanStack AI is the newest library in the TanStack ecosystem — from the team behind TanStack Query (React Query), Router, and Table. It launched in alpha on December 3, 2025, self-described as "a powerful, open-source AI SDK with a unified interface across multiple providers. No vendor lock-in, no proprietary formats, just clean TypeScript and honest open source."

The founding philosophy is direct vendor competition avoidance. Tanner Linsley explained on Reddit: "We saw enough improvement space on Vercel's solution that we wanted to build our own. One that is as close to our product tenets as possible. So far, that's resulted in better type-safety, better patterns around isomorphism, and honestly just the freedom to move in the direction we want without being beholden to another team."

The positioning is explicit: "The Switzerland of AI tooling" — neutral, transparent, and fully open-source. Sponsored by Cloudflare and Prisma, which have an incentive to support non-Vercel-dependent tooling.

#### Architecture

TanStack AI splits into a core package and provider adapters, similar to Vercel's design:

- `@tanstack/ai` — Core `chat()` function, tool definitions, agent loop strategies, streaming utilities
- `@tanstack/ai-openai`, `@tanstack/ai-anthropic`, `@tanstack/ai-google`, `@tanstack/ai-ollama`, `@tanstack/ai-openrouter` — Provider adapters
- `@tanstack/ai-react`, `@tanstack/ai-solid` — UI framework integrations
- `@tanstack/ai-client` — Vanilla JS client for any environment (terminal, browser, Node.js)
- `@tanstack/ai-devtools` — Built-in DevTools panel for real-time inspection

The central primitive is the `chat()` function, which takes an `adapter`, `messages`, and `tools`:

```typescript
import { chat, maxIterations } from '@tanstack/ai'
import { anthropicText } from '@tanstack/ai-anthropic'

const stream = chat({
  adapter: anthropicText('claude-opus-4-6'),
  messages: [{ role: 'user', content: 'Fix this bug' }],
  tools: [readFileTool, writeFileTool],
  agentLoopStrategy: maxIterations(10),
})

for await (const chunk of stream) {
  if (chunk.type === 'text') process.stdout.write(chunk.text)
}
```

#### Key Feature: Isomorphic Tools

This is the headline differentiator. TanStack AI's `toolDefinition()` lets you define a tool's schema once with Zod and then provide separate server or client implementations via `.server()` and `.client()`. You never duplicate the schema:

```typescript
const readFileDef = toolDefinition({
  name: 'readFile',
  inputSchema: z.object({ path: z.string() }),
  outputSchema: z.string(),
})

// Server implementation (runs during LLM tool call)
const readFile = readFileDef.server(async ({ path }) => {
  return fs.readFileSync(path, 'utf-8')
})

// Client implementation (runs in browser if needed)
const readFileClient = readFileDef.client(async ({ path }) => {
  return await fetch(`/api/file?path=${path}`).then(r => r.text())
})
```

With Vercel AI SDK, you're at 600 lines for ten tools. With TanStack AI, you're at 300 lines — and that's being conservative. The real savings come from eliminating the mental overhead of keeping two implementations in sync.

#### Key Feature: Per-Model Type Safety

The Vercel AI SDK uses flexible typing for provider options to enable rapid iteration on provider-specific features. The tradeoff is that you can pass options that don't apply to the model you're using, and TypeScript won't catch it. TanStack makes the opposite tradeoff, prioritizing per-model type safety. Zero runtime overhead since it's all erased at compile time.

When you specify `adapter: openaiText('gpt-4o')`, the TypeScript types narrow to only the options that `gpt-4o` supports. Passing reasoning flags meant for a different model will error at compile time, not at runtime at 2am.

#### Key Feature: Built-In DevTools

TanStack AI immediately ships with DevTools. Hell yes, why the AI SDK hasn't done this already boggles the mind. (Matt Pocock, TypeScript expert)

TanStack AI emits 35+ distinct event types, and the DevTools consume them all: chunk-level inspection, tool lifecycle tracking, token usage per message (cumulative and delta), and conversation replay — full session recording and playback.

#### Key Feature: AG-UI Protocol Support

TanStack AI supports the AG-UI protocol — an open standard for agent-to-frontend communication. This means a Python FastAPI or PHP Slim backend speaking AG-UI can drive TanStack AI's TypeScript client. Multi-language interoperability built in from day one.

#### Key Feature: Tool Approval Workflows

Built-in human-in-the-loop: sensitive tool calls (database writes, API calls with side effects) emit an approval event to the client before execution. The client approves or rejects, and the agent loop resumes accordingly. No custom middleware needed.

#### Strengths

The architectural clarity is genuine — defining tools once and deploying them with `.server()` / `.client()` is cleaner than Vercel's approach. Per-model type safety catches real mistakes at compile time. The DevTools are immediately useful for anyone debugging complex tool call chains.

The vendor-neutrality is structural, not just marketing. TanStack has no hosted service, no AI Gateway, no upsell path. The SDK routes directly to your chosen provider.

Its place in the broader TanStack ecosystem means Query/Router/Form/Store integrations are coherent by design. For teams already on TanStack, AI is a natural addition.

#### Weaknesses

**Alpha status is real.** TanStack AI launched in alpha on December 3, 2025. The team acknowledges "bugs, rough edges, things that don't work." Version 0.3.0 introduced breaking changes to AG-UI protocol events. For a production agent, this is a meaningful risk.

**Node.js v24+ is required** due to `isolated-vm` compatibility issues. Node.js v24+ is required to avoid compatibility issues with isolated-vm. Node.js v24 became LTS in October 2025 — most environments support it by April 2026, but it's still a harder constraint than Vercel's Node.js 18+ requirement.

**~10 providers vs. 30+ in Vercel.** The pure number of providers from the AI SDK is way beyond what TanStack can offer right now. If you need Bedrock, Azure OpenAI, Mistral, Perplexity, Fireworks, or 20 other providers, Vercel has them; TanStack does not yet.

**Less granular stream control.** The AI SDK gives you a ton more control over the stream with custom data parts than TanStack AI does right now.

**Small team, slower velocity.** Vercel has dedicated engineers; TanStack AI is volunteer-driven with Netlify/Cloudflare sponsorship. Feature parity will take time.

#### Terminal Agent Fit: ★★★☆☆

The core `chat()` function and `@tanstack/ai-client` work in Node.js terminals. But the Node.js v24 requirement, alpha stability, and limited provider support make it risky for a production agent today. The architecture is compelling; the execution is still maturing.

### The Verdict

**For building a terminal-based coding agent today: Vercel AI SDK (`ai`) is the clear choice.**

The reasoning is straightforward. OpenCode — the most-starred open-source JS/TS terminal coding agent — uses the Vercel AI SDK across all 120k+ installations. The pattern has been validated at production scale. The official Node.js quickstart guide demonstrates terminal streaming with `readline` and tool calling, with no framework required.

TanStack AI's isomorphic tools and per-model type safety are genuine architectural wins. But alpha status means real limitations: ~10 providers versus Vercel's 30+, less granular streaming control, and a small volunteer team means slower development pace. The Node.js v24+ requirement narrows compatibility. Breaking changes in the alpha will cost engineering time.

### The Long View (12–18 months)

"If you need production-ready features today, use the Vercel AI SDK. It's mature, well-documented, and battle-tested. But if you're thinking about the next two years? TanStack AI is the more interesting bet."

TanStack's trajectory matters. The team has a proven track record of shipping excellent, TypeScript-first, framework-agnostic libraries (Query → Router → Form → Table). Their philosophy has consistently won developer trust. If TanStack AI exits alpha with the same quality, the isomorphic tool system alone makes it worth adopting. Watch for the v1.0 milestone.

### Decision Framework

Use **Vercel AI SDK** if:
- You need to ship a production agent in the next 6 months
- You need more than 6 AI providers (Bedrock, Azure, Groq, Mistral, etc.)
- Node.js 18+ compatibility matters
- You want battle-tested patterns and 100+ code examples
- You're adopting the same stack as OpenCode, Gemini CLI, and Qwen Code

Use **TanStack AI** if:
- You're building a greenfield project and can absorb alpha risk
- Your team has >10 tools and the isomorphic pattern would cut code by 50%
- Per-model type safety is a hard requirement
- You're already on the TanStack ecosystem (Query, Router, etc.)
- You want built-in DevTools for debugging agent sessions
- Structural vendor neutrality matters (no Vercel account, ever)

Use **Both** via abstraction if:
- You want a migration path: build your agent's LLM interface as a thin adapter layer. Start with Vercel AI SDK today. When TanStack AI hits stable, swapping it out is a matter of changing the adapter — not rewriting your tools or agent logic.
